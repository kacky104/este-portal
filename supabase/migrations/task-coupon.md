# /mypage クーポン管理UIの追加

/mypage に、店舗オーナーが自分の店舗のクーポンを管理（追加・編集・削除・公開/非公開切替）できるUIを追加したい。

## 認証・オーナー→サロンの紐付け
- /mypage が既にオーナー認証＆担当サロンを特定している仕組み（今すぐ管理など既存機能と同じ）をそのまま再利用する。新たな認証方式は作らない。
- すべての操作は「ログイン中オーナーの自店舗（salon_id）」に対してのみ行う。他店舗のクーポンは一切操作・表示しない。

## /mypage のクーポン管理セクション（タブ）
- 既存 /mypage のレイアウト/タブの作法に合わせて「クーポン」セクション（またはタブ）を追加。
- 一覧：自店舗のクーポンを公開・非公開含めて表示（タイトル / 割引内容 / 条件 / 有効期限 / 公開状態）、sort_order 昇順。
- 新規追加フォーム：title（必須）/ discount（必須、例「¥1,000 OFF」）/ conditions（任意）/ valid_until（任意・日付入力）/ 公開・非公開トグル。sort_order は自動採番（既存最大+1）でよい。
- 各クーポンの編集（同じ項目）・削除（確認あり）・公開/非公開のワンタップ切替。
- データ書き込みは既存 /mypage の方式（クライアント or サーバーアクション）に合わせる。

## RLS の更新（マイグレーションSQLを作成、実行はユーザーが Supabase で行う）
- coupons に、オーナーが「自分の店舗の」クーポンを INSERT/UPDATE/DELETE できるポリシーを追加。スコープは厳密に「その行の salon_id がログインオーナーの担当サロンと一致する場合のみ」。他店舗は不可。
- オーナーは自店舗のクーポンを公開・非公開に関わらず SELECT できるようにする（/mypage 一覧用）。公開ページ用の「is_published=true は誰でも SELECT」既存ポリシーは維持。
- 既存の管理者UID用ポリシー・公開SELECTポリシーは壊さない。
- オーナー→サロンの判定は、既存の他のオーナー編集機能で使っているRLSの考え方に合わせる。

## スコープ/厳守
- 公開ページ /salon/[id]/coupon の表示は壊さない（読み取りは従来どおり）。
- /mypage の既存機能（今すぐ管理など）は一切壊さない。クーポンセクションの追加のみ。

## 進め方
作業前に、(a) /mypage の既存の認証・オーナー→サロン特定・データ書き込みの実装、(b) coupons の既存RLSポリシー、(c) 既存の「オーナーが自店舗データを編集する」RLSの書き方、を確認してから実装すること。
完了したら feedback_auto_deploy.md に従って自動でコミット→プッシュ→デプロイまで進めてOK。RLS更新のマイグレーションSQLはユーザーが Supabase の SQL Editor で実行する。
