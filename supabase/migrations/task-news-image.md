# お知らせに画像添付機能を追加

お知らせ（announcements）に画像を1枚添付できるようにしたい。オーナーが /mypage のお知らせフォームから画像をアップロードし、公開ページ /salon/[id]/news で表示する。既存の写メ日記（diary）の単一画像アップロードの実装を参考・踏襲して、同じ作法で作る。

## DB
- announcements に image_url text（NULL可）列を追加（マイグレーションSQLファイルを作成。実行はユーザーが Supabase SQL Editor で行う）。

## ストレージ
- お知らせ画像用の Supabase Storage バケットを用意する。既存の diary-images と同じ方針（公開読み取り、オーナーが自店舗分をアップロード）で、新規バケット（例：announcement-images）を作るか既存パターンに合わせる。
- バケット作成やストレージのRLS/ポリシーに必要なSQL・手順も出力し、実行はユーザーが行う（dashboard か SQL Editor）。

## /mypage（画像アップロード）
- お知らせの新規追加・編集フォームに「画像」アップロード欄を追加（任意・1枚）。
- 画像を選択するとストレージにアップロードし、その URL を image_url に保存。差し替え・削除（画像を外す）もできるようにする。
- 実装は既存の写メ日記（diary）の単一画像アップロードの方式に合わせる。

## 公開ページ /salon/[id]/news
- お知らせに image_url があれば、各お知らせのカードに画像を表示する（無ければ従来どおりテキストのみ）。
- レイアウトはサイトのテーマに統一。モバイルでも崩れないこと。

## スコープ/厳守
- 既存のお知らせ表示・/mypage のお知らせ管理・他機能は壊さない。画像対応の追加のみ。
- 画像が無い既存お知らせは従来どおり表示されること。

## 進め方
作業前に、(a) 写メ日記（diary）の画像アップロード実装（バケット・アップロード処理・表示・ストレージRLS）、(b) /mypage のお知らせフォーム、(c) announcements の既存スキーマ、を確認し、それに合わせて実装すること。
完了したら feedback_auto_deploy.md に従って自動でコミット→プッシュ→デプロイまで進めてOK。DB列追加・ストレージ関連のSQLはユーザーが Supabase で実行する。
