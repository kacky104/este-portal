# タスク：会員ダッシュボード /member の新規作成（フェーズ1：器を作る）

## 目的
会員（一般ユーザー）向けのホーム画面 /member を新規作成する。今は保存機能（/saved）が単独で存在するだけなので、その上位の「会員のホーム」を1枚用意し、今後の機能拡張（プロフィール・閲覧履歴・通知）の受け皿にする。

**フェーズ1のスコープは「器」を作ること。** 新しいDBテーブルは作らず、既存の saved_items のデータだけで構成する。プロフィール・閲覧履歴・通知は「準備中」のプレースホルダーとして枠だけ置く（中身はフェーズ2以降）。

## 重要な前提
- **/mypage はオーナー専用ページ。今回作る会員ダッシュボードは別物。URLは /member。** /mypage には一切手を触れない。
- 会員ログインは既存の Supabase Auth（/login）。/member は**ログイン必須**にする（未ログインなら /login へリダイレクト）。
- 保存データは既存の saved_items テーブル＋既存の保存抽象化レイヤー（saveStore.ts / saveStore 関連）を使う。新しい保存の仕組みは作らない。
- ブランド色：サロン＝ピンク #EC4899（肉球アイコン）、セラピスト＝紫 #A855F7（桜アイコン）、ロゴ・見出し＝オレンジ→マゼンタ #FB923C→#DB2777。既存のアイコン・配色定義があればそれを唯一のソースとして参照する。

## ルール
- 新規ページ追加（UI）なので、feedback_auto_deploy.md の手順で自動コミット→プッシュ→デプロイまで進める。
- DBスキーマ変更なし。環境変数の変更なし。
- 既存ページ（/saved, /mypage, トップ等）の挙動は変えない。

## 画面構成（モバイル基準・PCは中央寄せ＋最大幅で自然に）

新規ファイル `src/app/member/page.tsx` を作成。上から以下の順で構成する。

### 1. ヘッダー
- 既存の共通ヘッダー（ロゴ、保存数バッジ、アカウント/ログアウト）を流用する。トップや他ページと同じヘッダーコンポーネントを使い、見た目を統一する。

### 2. 挨拶エリア
- 「ようこそ」（小さいグレー文字）＋「マイページ」（大きい太字）。
- ログイン中の会員のメールアドレスを下に小さく表示（Supabase Auth のセッションから取得）。

### 3. 保存サマリー（2カード横並び）
- 左：保存したサロン数。ピンクの肉球アイコン＋「保存したサロン」＋件数（大きく #DB2777）＋「見る →」。タップで /saved のサロンタブへ遷移。
- 右：保存したセラピスト数。紫の桜アイコン＋「保存したセラピスト」＋人数（大きく #9333EA）＋「見る →」。タップで /saved のセラピストタブへ遷移。
- 件数は saved_items から種別ごとにカウント。
- /saved のタブ指定遷移：既存の /saved がタブ切替を URL クエリ等で受けられるなら活用する。無ければ単に /saved へ遷移で可（深掘りしすぎない）。

### 4. 最近保存したサロン（プレビュー）
- 見出し「最近保存したサロン」（左にピンクの縦バー）。
- 保存済みサロンのうち最近の数件（3〜4件程度）を横並びカードでプレビュー。サロン画像・名前・エリア。タップでそのサロン詳細（/salon/[id]）へ。
- 保存が0件のときは、このセクションは出さないか、「まだ保存がありません」の控えめな表示にする。

### 5. 最近保存したセラピスト（プレビュー）
- 見出し「最近保存したセラピスト」（左に紫の縦バー）。
- 保存済みセラピストのうち最近の数件を丸い顔写真＋名前でプレビュー。タップでそのセラピスト詳細（/therapist/[id]）へ。
- 0件のときの扱いはサロンと同様。

### 6. これからの機能（準備中プレースホルダー）
- 見出し「これからの機能」（左にグレーの縦バー）。
- 以下3つを、点線枠の無効化されたカードとして並べる。各カードにタイトル＋短い説明＋右端に「準備中」のグレーピル。クリックしても今は何も起きない（disabled 相当）。
  - プロフィール編集：「ニックネーム・アイコンを設定」
  - 閲覧履歴：「最近見たサロン・セラピスト」
  - 通知・新着フォロー：「保存サロンの新着・出勤を通知」
- これらはフェーズ2以降で実装するため、今は表示のみ。

## データ取得の方針
- ログインユーザーの取得とログインガードはサーバー側で行う（Supabase Auth セッション）。未ログインなら /login へ redirect。
- 保存データ（saved_items）の読み出しは既存の保存レイヤー（saveStore 等）と整合する形で行う。サロン・セラピストの表示用情報（名前・画像・エリア等）は、保存された id をもとに salons / therapists テーブルから取得する。
- 表示用クエリは互いに独立なものを Promise.all でまとめる（サロン情報とセラピスト情報の取得など）。
- このページはログインユーザー個別の内容なので ISR キャッシュ対象にはしない（revalidate は付けない／動的のままで良い）。トップやサロン詳細とは性質が違う点に注意。

## 導線（任意・軽め）
- 既存のアカウントメニュー（AccountMenu）に「マイページ（/member）」への導線を1つ足せると親切。ただし既存メニューの構造を壊さない範囲で。難しければ今回は省略可。
